所有权
let a = Box::new([0;1_000_000])；//a拥有这个box
let b=a;box的所有权由a交给了b
如果一个变量拥有一个box，当rust释放该变量的frame时，rust会释放该box的堆内存
变量被移动后，就不再允许使用
fn main(){
	let first = String::from("ferris");
     let full = add_sufix(first);//first的堆数据所有权被移动
     println!("{first}"); //这里的first无法再使用
}
fn add_suffix(mut name: String)-> String{
	name.push_str("Jr");
	name
}
rust在进行编译时检测时，尽管有永假判断（程序执行时不会用到），但rust仍然会检测里面的行为是否合法
未定义行为：使用未定义或使用未初始化变量，或者悬空指针和数组越界

1.在 move_a_box 之后通过 println 读取 b 是未定义的行为，因为它读取了已释放的内存。
2.给 b 赋予第二个所有者属于未定义行为，因为这会导致 Rust 代表 b2 第二次释放该箱。无论 let b2 = b 绑定发生在 move_a_box 之前还是之后，这都无关紧要。

let b=Box::new(0);
let b2 = b ;
println!({b});
move_a_box({b2});//3.这里尽管 b 被移动了，但其数据直到最后调用 move_a_box 时才被释放。因此这个程序从技术上讲是安全的，尽管仍然被 Rust 拒绝。

变量对数据有三种权限：
读（R）：数据可以被复制到另一个位置
写（W）：数据可以被修改

默认拥有（RO）

引用和借用（2）
权限是定义在位置上的（不仅仅是变量）
**位置**是任何可以放在赋值语句左侧的东西
位置包括：
1.变量，例如a
2.位置的解引用，例如 * a
3.位置的数组访问，例如a[0]
4.位置的字段访问，例如a.0(针对元组)a.field(针对结构体)
5.上述任何组合

要么多个不可变引用(共享引用)：只读的
要么一个可变引用(独占引用)：在不移动数据的情况下，临时提供可变访问，不允许原变量访问数据（移除了原变量所有权限）

当位置变成不使用的时候会失去权限，（有些权限是互斥的）因为如果有引用，引用使用这个位置指向的内容时，此时如果位置有权限去修改，会发生未定义行为。
```rust
let mut x = 10;
let r = &mut x;
- 在 `r` 生命周期内
- `x` 暂时“失去权限”
println!("{}", x); // 错误：x 暂时失去访问权限
```

当同时对一个数据有可变引用和不可变引用时，可变引用临时降级为不可变引用，因为这时不可变引用也可以访问那个数据，相当于有了别名，同一数据不应同时“被别名引用”和”具有可变性“。
```rust
let mut v:Vec<i32>=vec![1,2,3];
let num: &mut i32=&mut v[2];//*num R+W
let num2: &i32 =&*num;//*num R
					//*num2 R
println!("{*num}{*num2}");
```

1.权限在引用生命周期结束时被返回
2.数据必须在其所有的引用存在的期间存活

**流动权限F**：在表达式使用输入引用（表达式需要使用函数的输入参数）或函数返回输出引用时需要

F权限在函数体内不会发生变化

如果一个引用被允许在特定表达式中使用（即流动），那么它就具有F权限
